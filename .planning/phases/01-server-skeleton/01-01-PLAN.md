# Plan 01-01 — TypeScript + MCP SDK bootstrap

**Phase:** 1 (Server skeleton + install)
**Status:** Ready to execute
**Depends on:** Nothing (first plan of first phase)

## Goal

Stand up the package.json + tsconfig + src/ skeleton + vitest config + bin entrypoint stub. After this plan ships, `npm install && npm run build && node dist/index.js --version` works. Other Phase 1 plans (01-02, 01-03) depend on this scaffolding existing.

## Acceptance criteria

1. `package.json` declares `vaultpilot-mcp` v0.0.0 with bin entry `vaultpilot-mcp` → `dist/index.js`; runtime dep on `@modelcontextprotocol/sdk`; devDeps on `typescript`, `@types/node`, `vitest`, `tsx` (for dev/test)
2. `tsconfig.json` strict mode, target `ES2022`, module `NodeNext`, outDir `dist`
3. `src/index.ts` is the bin entrypoint with `#!/usr/bin/env node` shebang; routes `--check` vs default server boot via a single `if (process.argv.includes('--check'))` (the actual subcommand impl lands in 01-03, this is just the routing stub)
4. `src/server.ts`, `src/diagnostics/check.ts`, `src/tools/index.ts`, `src/config/index.ts` exist as stubs (functions throw `not yet implemented`)
5. `vitest.config.ts` exists with default node-environment config
6. `npm run build` produces `dist/` with the shebang preserved
7. `npm test` runs zero tests cleanly (no test files yet — vitest just exits)
8. `LICENSE` file (BUSL-1.1, mirroring upstream)
