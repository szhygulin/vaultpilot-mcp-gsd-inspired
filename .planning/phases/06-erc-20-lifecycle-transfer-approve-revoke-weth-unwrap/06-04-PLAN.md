## `src/protocols/weth9.ts` exact shape (LOCKED in this plan)

```typescript
// src/protocols/weth9.ts (NEW)
//
// WETH9.withdraw(uint256) — the one function Phase 6 uses. Research § Topic 2:
// viem does NOT ship a weth9Abi const; `parseAbi` is the canonical pattern
// for "I need this one method, not the whole interface."

import { type Address, type Hex, encodeFunctionData, parseAbi } from "viem";

import { getWethAddress, type ChainId } from "../config/contracts.js";

export const WETH9_WITHDRAW_ABI = parseAbi([
  "function withdraw(uint256 amount)",
  "function deposit() payable",  // Not used in v1.1 (wrap is v2+); included for
                                  // symmetry so a future plan that adds prepare_weth_wrap
                                  // doesn't need to evolve this const.
]);

// Selector const — preview_send.ts and downstream consumers reference this
// rather than inlining "0x2e1a7d4d". Format-fanout-sentinel.
// Verified via keccak256(toHex("withdraw(uint256)")).slice(0, 10) === "0x2e1a7d4d".
export const WETH9_SELECTORS = {
  withdraw: "0x2e1a7d4d" as Hex,
} as const;

// WETH decimals — constant across all chains (the contract is hard-coded to
// 18 decimals). Plan 06-04 uses this directly rather than going through
// get_token_metadata at prepare time; saves one RPC round-trip on the hot
// path. The registry has it too (it's in the top-50), but this avoids the
// registry dependency for prepare_weth_unwrap.
export const WETH9_DECIMALS = 18;

/**
 * Encode `WETH9.withdraw(amount)` calldata. Returns the 36-byte calldata
 * (4-byte selector + 32-byte amount). `tx.to` MUST be the WETH9 contract
 * address (use `getWethAddress(chainId)` from `src/config/contracts.ts`).
 */
export function encodeWethWithdraw(amount: bigint): Hex {
  return encodeFunctionData({
    abi: WETH9_WITHDRAW_ABI,
    functionName: "withdraw",
    args: [amount],
  });
}

/**
 * Convenience: returns the canonical WETH9 address for the given chain.
 * Re-exports from `src/config/contracts.ts` SOT — callers can import from
 * either location, but the SOT is `src/config/contracts.ts`.
 */
export function getWethContractAddress(chainId: ChainId): Address {
  return getWethAddress(chainId);
}
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Agent → `prepare_weth_unwrap` | ONE agent-supplied string (`amount`); `parseAmountStrict(amount, 18)` is the off-by-decimal guard. `decimals=18` is a constant (WETH9's hardcoded decimal precision), so no registry/RPC dependency — the strict check fires synchronously at the agent boundary. |
| `tx.to` resolution → SOT | `tx.to = getWethAddress(1)` is a deterministic lookup against the regression-tested `src/config/contracts.ts` SOT. Agent does NOT supply a contract address; tampering at the agent boundary cannot redirect the call to a malicious clone contract. |
| `LEDGER NOTICE` → user expectation | The block is a non-cryptographic UX layer. The trust anchor remains the device hash match (Fixture F `payloadFingerprint` + presign hash). NOTICE prose tells the user how to ENABLE blind-sign; the user then verifies the hash on-device. Tamper-evident: prose is server-side static; agent re-emission would diverge from the documented text. |
