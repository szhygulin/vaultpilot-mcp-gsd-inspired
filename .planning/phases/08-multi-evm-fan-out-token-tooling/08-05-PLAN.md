<objective>
Widen the WalletConnect v2 session manager from single-chain (mainnet-only) to multi-chain (5 EVM chains) pairing. Two surgical edits in `src/wallet/session-manager.ts`: (1) drive `REQUIRED_NAMESPACES.eip155.chains` from `getConfiguredChainIds()` (Plan 08-05 new helper) replacing the hardcoded `["eip155:1"]`; (2) replace the multi-chain refusal in `sessionToStatus` (`"v1.x is mainnet-only"`) with multi-chain accounting — `accountsByChain` Map + `activeChainId` selection + `partiallyPaired` detection. Extend `LedgerStatus` interface with 3 new fields + preserve 5 existing fields (back-compat aliases). Wire per-chain scope into `set_active_account` (Plan 08-02 added the optional chain arg; Plan 08-05 widens the body to consume `accountsByChain`).

Purpose: research § Topic 8 + the physical-device-testing report in ROADMAP (Plan 08-05 note line 35 of the roadmap) — Ledger Live's pairing UI surfaces accounts on every enabled chain, but ONLY when the wallet has the corresponding networks enabled in Ledger Live → Manage Accounts. The existing single-chain refusal in `session-manager.ts:620-632` ("v1.x is mainnet-only") was a Phase 3 deliberate guard — surfaced during physical-device testing as the bottleneck preventing cross-chain pairing. Plan 08-05 removes the refusal, adds multi-chain accounting, and surfaces the `partiallyPaired` flag (T-WC-PARTIAL-1) when the user's Ledger Live config doesn't cover all configured chains. The pair-flow tools themselves (`pair_ledger_live`, `pair_ledger_live_start`, `pair_ledger_live_wait`) stay byte-frozen — multi-chain support is automatic once `REQUIRED_NAMESPACES` reflects the configured set.

Output: 3 source MODIFY (`session-manager.ts` two surgical edits + LedgerStatus widening; `set_active_account.ts` per-chain scope body; `env.ts` +1 helper) + 1 new test file (`test/session-manager.multi-chain.test.ts`) + 2 test MODIFY (`test/set-active-account.test.ts` per-chain scope cases; `test/get-ledger-status.test.ts` multi-chain assertions + secret-safety). Single atomic commit: `feat(08-05): WalletConnect multi-chain pairing — REQUIRED_NAMESPACES from configured chains + sessionToStatus multi-chain accounting + partiallyPaired flag + set_active_account per-chain scope`.
</objective>
