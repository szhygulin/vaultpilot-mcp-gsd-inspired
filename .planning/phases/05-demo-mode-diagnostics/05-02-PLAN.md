## Persona-address null-defense (Q-AUTO-DEMO-PERSONA-DEFAULT + WRONG_MODE branch)

Under Plan 05-01's auto-demo seed, `getActivePersona()` is non-null in auto-demo mode by construction. Under explicit `VAULTPILOT_DEMO=true` (env arm) with NO prior `set_demo_wallet` call, `getActivePersona()` is null. The three tools defend defensively:

```typescript
// Pattern, applied identically in all three tools' demo branches:
if (isDemoMode()) {
  const persona = getActivePersona();
```

## `payloadFingerprint` cryptographic-binding regression value (KEY insight)

PREP-03's preimage is `chainId || to || valueWei || data` — `from` is NOT in the preimage. **Fixture A's hex anchor `0x7e1867b2e6bc98cbce57bb901a33e973c749565eb19f8b86056197c7a20b2f5a` holds REGARDLESS of whether `from` is the paired Ledger address (Phase 4 test) or the persona address (Phase 5 demo test).** This is the cryptographic-binding chain's regression value: same inputs (`to`, `valueWei`, `chainId`, `data`) → same fingerprint, independent of `from`.

The demo-flow integration test asserts this byte-identity: for `prepare_native_send({ to: "0x70997970C51812dc3A010C7d01b50e0d17dc79C8", valueWei: "1000000000000000000" })` under demo mode with whale as the active persona, the response's `payloadFingerprint === "0x7e1867b2..."` (Fixture A) byte-for-byte. The from-independence is the load-bearing claim of this test.

## Preview-time presign hash under demo mode

The presign hash preimage includes `nonce` (PREP-04 — pinned at preview from RPC `getTransactionCount`). Under demo mode, `getTransactionCount` is called against the persona address — so the demo presign hash differs from Fixture C unless the integration test mocks the nonce/fees/gas to match Fixture C's exact values (chainId 1, nonce 7, gas 21000, maxFeePerGas 30 gwei, maxPriorityFeePerGas 1.5 gwei).

The integration test PINS the mocked RPC to Fixture C's exact values. This makes the demo presign hash byte-identical to Fixture C — the cryptographic-binding regression value holds end-to-end through the demo path. Asserted in the integration test's preview step.
