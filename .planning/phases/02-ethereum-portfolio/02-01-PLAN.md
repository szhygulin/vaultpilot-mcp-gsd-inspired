# Plan 02-01 — Ethereum RPC client + tool registration scaffold

**Phase:** 2 (Ethereum read-only portfolio)
**Status:** Ready to execute
**Depends on:** Phase 1 complete

## Goal

`viem`-backed Ethereum public client with PublicNode default + `ETHEREUM_RPC_URL` override. Every read produces a shape that can carry a `rpcDegraded` flag. `src/tools/register-all.ts` exists as the single import point all later tool modules will register through. After this plan, the foundation for all Phase 2 read tools is in place.

## Acceptance criteria

1. `viem ^2.x` added to `package.json` runtime deps; `npm install` + lockfile committed
2. `src/chains/ethereum.ts` exports `getEthereumClient()` returning a memoized viem `PublicClient` configured against `ETHEREUM_RPC_URL` if set, else PublicNode
3. `src/chains/ethereum.ts` exports `isPublicNodeFallback(): boolean` — true when the default fallback is in use
4. `src/config/env.ts` (replace stub) — type-safe accessors `getEthereumRpcUrl()`, `getRpcProvider()`, `getRpcApiKey()`; never throws; missing → `undefined`
5. `src/tools/register-all.ts` (new) — exports `registerAllTools()` (initially empty body); imports nothing yet
6. `src/server.ts` — modify `buildServer()` to call `registerAllTools()` once at boot
7. One-time stderr warning when PublicNode default is in use; deduped per process
8. `test/chains-ethereum.test.ts` covers: PublicNode default, custom URL override, dedup of the stderr warning
9. `npm run typecheck` + `npm run build` + `npm test` all pass

## Out of scope (defer)
