# Plan 02-02 — ERC-20 balance scanner + top-50 token registry

**Phase:** 2 (Ethereum read-only portfolio)
**Status:** Ready to execute
**Depends on:** 02-01 (uses `getEthereumClient()`)

## Goal

`scanErc20Balances(wallet, tokens?)` returns balance + decimals + symbol for every non-dust token via a single multicall round-trip. Ships the static top-50 token registry. After this plan, balance enumeration is solved; pricing + portfolio aggregation lands in 02-03.

## Acceptance criteria

1. `src/tokens/ethereum-top-50.json` exists with 50 entries: `{ address, symbol, decimals, name }` — addresses checksummed, sourced from a public coingecko/coinmarketcap snapshot, top-by-trading-volume on Ethereum mainnet
2. `src/tokens/registry.ts` exports `loadEthereumTokenRegistry(): Token[]` (just `JSON.parse` the file)
3. `src/chains/erc20-scanner.ts` exports `scanErc20Balances(wallet, tokens?): Promise<TokenBalance[]>` — defaults `tokens` to the top-50 registry; uses `viem.multicall` over `balanceOf(wallet)` for each token; returns `bigint` balances
4. Helper `formatTokenBalance(balance: bigint, decimals: number): string` — uses `viem.formatUnits` for decimal-string output at the API boundary
5. Filter helper `filterDust(balances, decimals, prices?, threshold)` — separated from the scanner so 02-03 can wire in USD-aware filtering
6. `test/erc20-scanner.test.ts` covers: empty wallet returns empty array; single token with non-zero balance returns one entry; multicall payload shape verified via mock transport
7. `npm run typecheck` + `npm run build` + `npm test` all pass
