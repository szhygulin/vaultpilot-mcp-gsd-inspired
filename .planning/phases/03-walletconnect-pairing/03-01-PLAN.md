<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP stdio (stdout protocol / stderr diagnostics) | Any byte WC SDK writes to stdout corrupts the MCP client; `logger: "error"` + stderr-only diagnostics is the load-bearing mitigation |
| Process CWD → filesystem | WC's default Node storage writes `walletconnect.db` to `process.cwd()`; we never want this — fresh process = fresh pair by design |
| Env var → singleton init | `WALLETCONNECT_PROJECT_ID` is the gate; missing must be surfaced once with an actionable message, not as "client not initialized" on every subsequent call |
| WC relay (HTTPS / WSS) | Out-of-scope for this plan; covered by TLS at transport; project-ID gating limits relay rate |
